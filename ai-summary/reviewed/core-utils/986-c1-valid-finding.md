## Trace Summary

`Spec.structToNative` at `src/contract/spec.ts:1147-1164` decodes Soroban UDT struct return values into plain JS objects. The method creates `res = {}` and iterates over struct fields, assigning decoded values with bracket notation: `res[field.name().toString()] = this.scValToNative(entry.val(), field.type())` (lines 1158-1161).

Field names come from the contract's WASM spec via `udt.fields()`. The `isNumeric` guard at line 1150 (`fields.some(isNumeric)`) only routes to the safe vec-return path when at least one field name is pure digits; `"__proto__"` contains non-digit characters and routes to the bracket-notation map path.

When `field.name().toString()` returns `"__proto__"`, the assignment `res["__proto__"] = decodedValue` triggers the `Object.prototype.__proto__` setter. If the decoded value is a JS object (from a nested struct or map type), `OrdinarySetPrototypeOf(res, decodedValue)` fires, changing the prototype of the returned object. Any properties on the new prototype are inherited by `res`. If the decoded value is `null` (from `scvVoid`), the prototype chain is severed entirely, causing `res.hasOwnProperty(...)`, `res.toString()`, etc. to throw TypeError.

Entry path: `funcResToNative` (spec.ts:612) → `Spec.scValToNative` (spec.ts:985) → `scValUdtToNative` (spec.ts:1098) → `structToNative` (spec.ts:1147). `funcResToNative` is the public API for decoding contract function return values.

Confirmed: `base/scval.ts:scValToNative` handles maps via `Object.fromEntries` (safe against `__proto__` via `CreateDataPropertyOrThrow`), and `Spec.scValToNative` handles typed maps by returning array-of-pairs (lines 1049-1053). The vulnerability is specific to `structToNative`'s bracket-notation assignment on a plain `{}` object.

## Findings

A contract published on-chain (or served by an attacker-controlled RPC) can include a UDT struct definition with a field named `"__proto__"`. When the SDK decodes a return value of that struct type, `structToNative` mutates the prototype chain of the result object:

1. **Object prototype injection**: If the `__proto__` field type is a struct or map type, the decoded value is a JS object. Its properties become inherited properties of the result. An application checking `result.isAdmin` or similar would see an attacker-injected `true` value via prototype lookup even though it is not an own property.

2. **Prototype chain severance**: If the `__proto__` field type is `void` or an Option resolving to null, `res.__proto__ = null` removes the entire prototype chain. Any subsequent code calling `res.hasOwnProperty()`, `res.toString()`, `Object.keys(res)`, or using `res` in string interpolation would throw TypeError, causing denial of service or error-path fallback behavior.

3. **No sanitization or safe-key check exists**: No call to `Object.defineProperty`, `Object.create(null)`, or key validation is performed before assignment. The `isNumeric` guard (spec.ts:40-42) is irrelevant since `__proto__` is not numeric.

Trust boundary: the contract spec is authored by third parties on-chain. A developer using an autogenerated contract client does not review every field name in the spec. The attack surface is "interact with any third-party Soroban contract that has a __proto__ struct field."

`__proto__` is a valid Rust identifier (not a keyword, starts with `_`), so a Soroban contract can legitimately define such a struct field.

## PoC Guidance

- **Test file**: `test/unit/contract_spec.ts` or new file in `test/unit/`
- **Setup**: Create a mock `xdr.ScSpecUdtStructV0` with two fields: `"name"` (string type) and `"__proto__"` (struct type with a field like `"isAdmin"` of bool type). Create a matching `xdr.ScVal` map with the corresponding entries.
- **Steps**: Call `spec.scValToNative(scv, typeDef)` where `typeDef` is a UDT pointing to the struct definition.
- **Assertion 1**: Verify `Object.getPrototypeOf(result) !== Object.prototype` — the prototype has been mutated.
- **Assertion 2**: Verify `result.isAdmin === true` despite `"isAdmin"` not being in `Object.keys(result)` — inherited property injection.
- **Assertion 3**: For the null case (void-typed `__proto__` field), verify that `result.hasOwnProperty` throws TypeError.
